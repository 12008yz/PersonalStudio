# Ручной чеклист: DPI 125% и 150% (PerMonitorV2)

В приложении заявлен режим **PerMonitorV2** в [`src/ScreenRecorder.App/app.manifest`](../src/ScreenRecorder.App/app.manifest) (`dpiAwareness` в `windowsSettings`). Автотесты CI не подменяют системный масштаб дисплея — проверка **вручную** на реальной машине.

---

## Подготовка

1. Соберите и запустите приложение как обычно (`dotnet run` из проекта App с нужной платформой или F5).
2. Убедитесь, что в **Диспетчере задач** нет явных аномалий до теста (опционально — базовая линия по памяти процесса).

---

## Масштаб 125%

1. **Параметры Windows** → **Система** → **Дисплей** → **Масштаб** → выберите **125%** (при нескольких мониторах проверьте основной монитор, на котором открыто окно приложения).
2. **Выйдите из сеанса и войдите снова** или перезапустите приложение, если ОС предлагает перезапуск после смены масштаба.
3. Запустите **Screen Recorder**, проверьте:
   - интерфейс без обрезанных подписей и «мыльных» несоответствий в типичных экранах;
   - кнопка **«Проверить захват экрана (10 с)»** завершается без необработанного исключения;
   - в логе активности: старт теста, затем строка результата с **ненулевым** числом кадров (на типичном ПК >> 0 за 10 с);
   - при желании — смените масштаб **во время** теста: допустимо появление предупреждения о неудачном `Recreate` пула (`CaptureTest_PoolRecreateFailures`) — это ожидаемый сценарий проверки смены DPI/разрешения.

**Критерий «ок»:** нет падения процесса; при стабильном масштабе 125% кадры идут, текст UI читаем.

---

## Масштаб 150%

Повторите те же шаги, выбрав **150%** вместо 125%.

**Критерий «ок»:** тот же, что для 125%.

---

## Несколько мониторов (опционально)

Если доступны два дисплея с **разным** масштабом (например, встроенный 150%, внешний 100%):

- перетащите окно приложения на каждый монитор и убедитесь, что масштаб интерфейса соответствует монитору (PerMonitorV2);
- тест захвата по умолчанию идёт на **основном** мониторе — при необходимости смените основной дисплей в параметрах и повторите тест.

## English (short)

**PerMonitorV2** is set in `app.manifest`. Manually set display scale to **125%**, run the in-app **10 s capture test**, expect **non-zero frames** and no crash; repeat at **150%**. Optional: mixed multi-monitor scaling. No CI substitute for changing system DPI.
